# 어드민 Analytics - 유저 여정 트래킹 분석

> 최종 수정일: 2026-03-24

---

## 한줄 요약

고객이 PLIC 사이트에서 **뭘 보고, 뭘 누르고, 어디서 나가는지** 자동으로 기록한 뒤,
어드민에서 한눈에 볼 수 있게 정리해주는 시스템.

---

## 데이터가 쌓이는 과정

```
고객이 사이트 방문/클릭/송금 등 행동
  → tracking.ts가 행동을 기록
  → 10개 모이거나 5초 지나면 서버로 전송
  → DynamoDB "plic-events" 테이블에 영구 저장
  → 어드민 > Analytics > 유저 여정 트래킹 탭에서 조회
```

---

## 수집/집계 원칙

| 역할 | 담당 | 설명 |
|------|------|------|
| **수집** | 클라이언트 (TrackingProvider + 페이지별) | 유저 행동은 브라우저에서만 감지 가능 |
| **집계/계산** | 서버사이드 (user-journey API) | 일관된 기준, 소급 적용, 중복 방지 |

---

## 자동으로 수집되는 것 (TrackingProvider)

TrackingProvider가 모든 고객 페이지에 깔려있어서, 아래 항목은 **코드 추가 없이 자동 수집**됨.

| 뭘 추적하나 | 언제 기록되나 | 예시 |
|------------|-------------|------|
| **페이지 방문** | 고객이 페이지를 이동할 때마다 (자동, 중복방지) | 홈→송금페이지 이동 |
| **스크롤 깊이** | 페이지를 25/50/75/100% 내렸을 때 | 랜딩페이지 50%까지 봄 |
| **체류 시간** | 사이트에 30초/1분/3분/5분/10분 머물 때 (마일스톤 이벤트) | 3분 이상 체류 |
| **탭 이탈/복귀** | 다른 탭으로 갔다가 돌아올 때 | 카톡 보다가 다시 옴 |
| **페이지 이탈** | 사이트를 떠날 때 (브라우저 닫기 등) | /deals/new에서 이탈 |
| **클릭** | `data-track` 속성이 붙은 버튼/링크 클릭 | CTA "무료로 시작하기" 클릭 |
| **섹션 노출** | `data-section` 속성 영역이 화면에 50% 이상 보일 때 | 랜딩 "보안" 섹션 노출 |
| **JS 에러** | 코드 에러 발생 시 | TypeError 등 |
| **페이지 로딩 속도** | 페이지 로드 완료 시 | 로드 1.2초 |

---

## 각 페이지에서 수동으로 추적하는 것

| 페이지 | 뭘 추적하나 |
|--------|------------|
| **회원가입** | 가입 시작 → 카카오 인증 → 정보 입력 → 사업자 정보 → 가입 완료 (5단계 퍼널) |
| **로그인** | 로그인 성공 / 실패 |
| **거래생성 (deals/new)** | 송금 시작 → 정보 입력 → 수취인 → 증빙 업로드 완료 → 거래 생성 (5단계 퍼널) |
| **결제** | 결제 진입 → 결제 시도 → 성공/실패 → 송금 완료(승인) |
| **랜딩** | 섹션별 노출, CTA 클릭 |
| **홈** | 페이지뷰, 메뉴 클릭 |

---

## 세션 관리

| 항목 | 방식 |
|------|------|
| 세션 생성 | 탭 열 때 sessionStorage에 생성 |
| 세션 만료 | **30분 비활성** 시 자동 갱신 (GA4 기준) |
| 익명 ID | localStorage에 영구 보관 (anonymousId) |
| 유저 식별 | 로그인 시 `tracking.identify(uid, role)` |

## 어드민에서 보이는 지표 설명

### 필터 기능

| 필터 | 설명 |
|------|------|
| **기간** | 전체 / 오늘 / 1주일 / 1개월 / 1년 / 직접 설정 |
| **운영 데이터 제외** | 기본 ON. 운영진 세션을 통째로 제외 |

### 운영진 필터 기준 (이중 방식)

**서버사이드 (API 조회 시):**
1. 운영진 UID(5개)로 로그인한 이력이 있는 세션 → 해당 세션 전체 제외
2. /admin 경로에 접근한 이력이 있는 세션 → 해당 세션 전체 제외
3. userRole이 'admin'으로 태깅된 세션 → 해당 세션 전체 제외

**클라이언트 (브라우저):**
- /admin 페이지 접근 시 localStorage에 마킹 → 이후 해당 기기의 모든 이벤트에 admin 태깅
- 로그인 전 활동(랜딩 방문 등)도 같은 기기면 자동 필터

### 서브탭 1: 전체 요약

| 지표 | 뜻 | 어떻게 계산하나 |
|------|-----|----------------|
| 총 이벤트 | 기간 내 발생한 모든 행동 기록 수 | 전체 이벤트 개수 |
| 세션 수 | 방문 횟수 (30분 비활성 시 새 세션) | 고유 sessionId 수 |
| 미가입 방문자 | 로그인 안 한 방문자 수 | 고유 anonymousId 수 |
| 가입 유저 | 로그인한 사용자 수 | 고유 userId 수 |
| 탭 이탈 | 다른 탭으로 간 횟수 | `tab_hidden` 이벤트 수 |
| 탭 복귀 | 다시 돌아온 횟수 | `tab_visible` 이벤트 수 |
| 이벤트 유형 분포 | pageview/click/funnel 등 비율 | eventType별 GroupBy |
| 디바이스 분포 | 데스크톱/모바일/태블릿 비율 | userAgent 문자열로 판별 |
| 페이지별 방문 수 | 어떤 페이지가 많이 보이나 (상위 20) | pageview 이벤트의 path별 카운트 |
| 이탈 페이지 | 어떤 페이지에서 사이트를 떠나나 (상위 10) | `page_exit` 이벤트의 path별 카운트 |
| UTM 유입 소스 | 어디서 들어왔나 (인스타/네이버 등) | URL의 utm_source 파라미터 |
| 일별 추이 | 기간 내 이벤트/세션 수 변화 | 날짜별 GroupBy |

### 서브탭 2: 퍼널 분석

4개 퍼널이 **독립적으로** 작동. 각 단계별 **유니크 세션 수** + 이탈률을 막대그래프로 표시.
(같은 세션에서 뒤로가기로 같은 단계를 여러 번 밟아도 1건으로 카운트)

| 퍼널 | 단계 |
|------|------|
| 회원가입 (2026.03.24~) | 가입 시작 → 카카오 인증 → 정보 입력 → 사업자 정보 → 가입 완료 |
| 로그인 | 성공 / 실패 |
| 거래생성 | 시작 → 정보 → 수취인 → 증빙 업로드 완료 → 거래 생성 |
| 결제 | 결제 진입 → 시도 → 성공 / 실패 / 송금 완료(승인) |

> 예: 기존 회원이 로그인 후 기존 거래 결제만 하면 → 로그인 + 결제 퍼널에만 데이터 찍힘

### 서브탭 3: 랜딩 행동 분석

**랜딩 페이지(/landing) 전용 데이터**만 집계.

| 지표 | 뜻 | 데이터 소스 |
|------|-----|------------|
| 랜딩 스크롤 깊이 | 랜딩 페이지를 얼마나 내려보나 (25/50/75/100%) | 서버 집계 (path=/landing 필터) |
| 체류 시간 마일스톤 | 세션별 체류 시간 구간 분포 (누적 방식) | **서버사이드 타임스탬프 기준** |
| 랜딩 섹션 노출 | 랜딩 각 섹션(hero/features/security 등) 실제 노출 수 | 서버 집계 (path=/landing 필터) |
| 랜딩 CTA 클릭 | 랜딩의 CTA 버튼 클릭 수 | 서버 집계 (path=/landing 필터) |
| 전체 클릭 현황 | 전 페이지의 data-track 클릭 | 전체 click 이벤트 |
| 에러 현황 | 발생한 JS 에러 (상위 10) | 전체 error 이벤트 |

**체류 시간 구간 (누적 방식):**
- 10초 미만 / 10~30초 / 30초 이상 / 1분 이상 / 3분 이상 / 5분 이상 / 10분 이상
- "30초 이상"은 1분 이상, 3분 이상 등을 모두 포함
- 서버에서 세션별 첫/마지막 이벤트 타임스탬프 차이로 계산 (데이터 소스 통일)

### 서브탭 4: 이벤트 기록

- 최근 50건의 **원본 이벤트 로그** (시간, 유형, 이벤트명, 페이지, 유저구분)
- 유저구분: 운영/가입자/미가입 3종으로 표시
- **CSV 다운로드** 가능 (엑셀로 열 수 있음, 한글 BOM 포함)

## 이벤트 안정성

| 항목 | 방식 |
|------|------|
| 전송 | sendBeacon (페이지 이탈 시에도 보장) + fetch fallback |
| 버퍼링 | 10건 또는 5초마다 전송 |
| 버퍼 제한 | 최대 100건 (무한 누적 방지) |
| 이벤트 검증 | 클라이언트: eventType/필수필드/퍼널스텝 검증. 서버: 타입/길이/100건 제한 |
| 프로덕션 로그 | 개발 모드에서만 console.log 출력 |
